# Scope Intake Template

Use this checklist before running a security review.

## Required

- Authorized target or repository path
- Surface type: web, API, mobile, repo, IaC, pipeline, or mixed
- In-scope components or directories
- Explicit exclusions
- Environment constraints

## Useful Clarifiers

- Authentication or privileged flows involved
- Sensitive data classes handled
- Third-party services or secrets managers in use
- Compliance or framework mapping requested
- Whether network access is permitted

## Output Expectations

- Quick triage only, or full report
- Engineering-focused remediation vs stakeholder summary
- Machine-readable findings needed or not
